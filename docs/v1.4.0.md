# v1.4.0 Release Notes

**Released:** 2026-03-17
**Tag:** v1.4.0
**Previous:** v1.3.0

---

## Summary

v1.4.0 is the largest release to date. It adds the Chrome extension, responsive desktop layout, and a suite of backend hardening changes. The system prompt was substantially refined and the paragraph rendering model was upgraded to semantic HTML.

---

## What Changed

### Chrome Extension

A fully functional Manifest V3 Chrome extension was added:

- Right-click any selected text → **Muunna selkokielelle** to translate
- `Alt+S` keyboard shortcut for instant translation
- Result displayed in a 400px Shadow DOM sidebar panel that slides in from the right
- 4 states: loading (skeleton shimmer), result, error, and empty-selection toast
- Copy to clipboard with visual confirmation
- Escape key, click-outside, and X button to dismiss
- Slow-network hint after 4 seconds

Files: `extension/manifest.json`, `extension/background.js`, `extension/content.js`, `extension/content.css`, `extension/icons/`

### Responsive Layout

Added `@media (min-width: 800px)` breakpoint to `frontend/index.html`:

- Side-by-side CSS grid layout: input on left, output on right
- Max width expanded to 1080px
- Arrow separator hidden at desktop sizes
- Textarea height increased to 400px on desktop

### Extension Landing Page

New `frontend/laajennos.html` with:

- What the extension is and how it works
- Step-by-step dev mode installation guide
- Usage instructions, limitations, and privacy information
- Footer links added to `index.html` and `tietosuoja.html`

### System Prompt v3.1

The system prompt was substantially rewritten:

- Strengthened injection defense: questions, commands, and meta-instructions are explicitly rejected
- Added paragraph guidance: separate ideas with double newlines
- Added list guidance: use `-` or `•` for enumerations
- Added heading guidance: use sparingly, only when they clarify structure
- Added number/date formatting: spell out 1–11, Finnish date format (`14.3.2026`)
- Expanded vocabulary examples (slang substitutions)
- Reader framing rules clarified

### Paragraph Rendering

Result text is now rendered as semantic HTML `<p>` elements instead of `white-space: pre-wrap`. Implemented in both `frontend/index.html` and `extension/content.js`. XSS-safe: text is escaped via `textContent` before injecting as `innerHTML`.

### Input Muting

After translation, the input textarea is visually muted to direct the user's focus to the output. Implemented in `frontend/index.html`.

### Backend Hardening

- **Startup assertions:** `assert OPENROUTER_API_KEY` and `assert ALLOWED_ORIGIN` — service fails fast on missing credentials
- **`MODEL` env var:** model is now configurable at runtime; defaults to `openai/gpt-4o-mini`
- **`max_tokens` reduced:** 4000 → 2500 (plain Finnish translations rarely exceed 800 tokens)
- **Rate limiting:** `@limiter.limit("30/hour")` via `slowapi`; returns HTTP 429 with Finnish error message
- **Health endpoint:** `GET /api/health` returns `{"status": "ok"}`
- **`EXTENSION_ORIGIN` support:** CORS list now conditionally includes extension origin

### `.gitignore`

Added `*.key`, `*.pem`, `*.env` entries to prevent accidental commits of secrets.
