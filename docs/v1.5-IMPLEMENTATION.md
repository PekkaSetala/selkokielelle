#### 1.3 VPS Configuration (Manual Steps)

**Nginx Configuration:**
Location: `/etc/nginx/sites-enabled/selkokielelle` (or your config)

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $remote_addr;  # ← ADD THIS LINE
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

**Why:**
- `slowapi` rate limiter uses `get_remote_address()` which checks `X-Forwarded-For` header first
- Without this, all requests appear from `127.0.0.1` (the server itself)
- Result: rate limiting doesn't work per-client, just per-server
- With this: each user's IP is tracked separately for rate limits

**Systemd Configuration:**
Location: `/etc/systemd/system/selkokielelle.service`

```ini
[Service]
ExecStart=/var/www/selkokielelle/backend/venv/bin/uvicorn \
    main:app \
    --host 127.0.0.1 \
    --port 8000 \
    --proxy-headers \
    --forwarded-allow-ips='127.0.0.1'
```

**Why:**
- `--proxy-headers`: tells FastAPI to trust proxy headers
- `--forwarded-allow-ips='127.0.0.1'`: trusts `X-Forwarded-For` from Nginx (on localhost)
- Without these: FastAPI ignores proxy headers, so rate limiter still doesn't work

**Why:**


## Section 4: Paragraph Rendering

### What Was Implemented

#### 4.1 Website Paragraph Rendering
**File:** `frontend/index.html` (lines 411-417)

```javascript
function renderParagraphs(text) {
  return text.split(/\n\n+/).map(para => {
    const div = document.createElement('div');
    div.textContent = para.trim();
    return `<p>${div.innerHTML}</p>`;
  }).join('');
}
```

**How it works:**
1. Split result text on `\n\n+` (one or more double newlines)
2. For each paragraph:
   - Create a temporary `<div>`
   - Set its `textContent` (safe, auto-escapes HTML)
   - Extract `innerHTML` (now safely escaped)
   - Wrap in `<p>` tags
3. Join all `<p>` elements

**Why this approach:**
- **Safe:** `textContent` prevents XSS (HTML injection)
- **Semantic:** Each paragraph is a real `<p>` element
- **Clean:** No `white-space: pre-wrap` hacks

**CSS Changes:**
```css
#output-text {
  /* Removed: white-space: pre-wrap; */
  opacity: 0;
  transition: opacity .35s ease;
  width: 100%;
}

#output-text p {
  margin: 0 0 1em 0;        /* 1em gap between paragraphs */
}

#output-text p:last-child {
  margin-bottom: 0;         /* No gap after last paragraph */
}
```

### Why This Approach

**Mobile-first CSS:**
- Base styles work on mobile (single column)
- Desktop breakpoint enhances experience
- No mobile-breaking changes on large screens

- Clean layout without flexbox nesting
- Easy to position full-width elements (header, footer)
- Responsive without media queries for each element

**Side-by-side UX:**
- Input and output visible simultaneously
- No scrolling to see both
- Faster workflow (especially important for accessibility users)

**Rotated arrow:**
- Visual indicator that layout changed
- Points horizontally between columns
- Still shows connection between input and output
